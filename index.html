🔴 Hiện trạng (AS-IS)

1

Nhận hồ sơ giấy / scan

CMND, sổ hộ khẩu, bảng lương, sao kê ngân hàng, CIC… đều ở dạng ảnh/PDF

2

Nhập liệu thủ công

Nhân viên gõ tay thông tin từ hồ sơ vào core system → sai sót, chậm trễ

3

Đối chiếu rule bằng tay

Check chéo hàng chục điều kiện (DTI, thu nhập tối thiểu, lịch sử nợ…) → dễ bỏ sót

4

Trình duyệt nhiều cấp

Hồ sơ chuyển tay qua 3-4 cấp duyệt → tắc nghẽn, không theo dõi được trạng thái

5

Giải ngân chậm

Trung bình 3-7 ngày/hồ sơ, peak season có thể lên 2 tuần

🟢 Mục tiêu (TO-BE)

1

Upload/Scan tự động

Hồ sơ upload qua app/portal → OCR tự trích xuất toàn bộ data trong vài giây

2

Auto-fill & Cross-check

Data tự điền vào hệ thống, AI đối chiếu chéo giữa các loại giấy tờ → phát hiện bất nhất

3

Rule Engine tự động

100% rule hiện hành được mã hóa → hệ thống tự check, phân loại Xanh/Vàng/Đỏ

4

AI Risk Scoring

ML model chấm điểm rủi ro, auto-approve hồ sơ Xanh, route Vàng/Đỏ cho specialist

5

Giải ngân nhanh

Hồ sơ Xanh: <30 phút. Vàng: <4 giờ. Đỏ: <24 giờ (cần specialist review)

⚠️ Rủi ro cần lưu ý
Rule xét duyệt có thể không được document đầy đủ — nhiều rule nằm trong "kinh nghiệm" của nhân viên senior. Cần dành đủ thời gian phỏng vấn sâu và cross-validate giữa các chi nhánh. Data lịch sử có thể thiếu label chính xác (lý do reject không rõ ràng) → cần effort bổ sung để re-label.

Phase 1 — OCR Engine & Document Processing
Xây dựng engine OCR trích xuất dữ liệu từ hồ sơ (CMND/CCCD, bảng lương, sao kê ngân hàng, báo cáo CIC), auto-classify loại giấy tờ, validate dữ liệu đã trích xuất.
8–10 tuần AI/ML Engineer + Backend Dev Core
Document Classifier
AI
Model phân loại tự động loại giấy tờ khi upload: CMND mặt trước/sau, CCCD, bảng lương, sao kê NH, hợp đồng lao động, xác nhận CIC. Dùng CNN hoặc Vision Transformer, target accuracy >98%.
2–3 tuần
OCR trích xuất CMND/CCCD
OCR
Trích xuất: họ tên, ngày sinh, giới tính, số CMND/CCCD, ngày cấp, nơi cấp, địa chỉ thường trú. Xử lý ảnh chụp nghiêng, mờ, ánh sáng kém. Cross-check với eKYC face matching nếu có.
2–3 tuần
OCR trích xuất bảng lương & HĐLĐ
OCR
Trích: lương gross/net, phụ cấp, khấu trừ, ngày trả lương, tên công ty, mã số thuế. Xử lý đa dạng format bảng lương (mỗi công ty 1 kiểu). NLP hỗ trợ nhận diện field khi layout thay đổi.
3–4 tuần
OCR trích xuất sao kê ngân hàng
OCR
Trích: danh sách giao dịch (ngày, mô tả, số tiền, số dư), tổng thu nhập/chi tiêu hàng tháng. Nhận diện pattern: lương đều đặn, nguồn thu bất thường, dòng tiền âm liên tục.
3–4 tuần
Data Validation & Cross-check
AI
So sánh chéo dữ liệu giữa các giấy tờ: tên trên CMND = tên trên bảng lương? Thu nhập trên bảng lương khớp với dòng tiền sao kê? Phát hiện bất nhất tự động, flag cho reviewer.

Infra

Phase 2 — Rule Engine & Traffic Light System
Mã hóa toàn bộ quy tắc xét duyệt vào hệ thống, xây logic phân loại đèn giao thông (Xanh/Vàng/Đỏ), auto-routing hồ sơ theo kết quả đánh giá rule.
6–8 tuần Backend Dev + BA + Risk Core
Hard Rule Implementation
Rule
Mã hóa các rule bắt buộc: tuổi (18–60), thu nhập tối thiểu, DTI ratio ≤ ngưỡng, không nằm trong blacklist, CIC group nợ ≤ 2, thâm niên công tác ≥ 6 tháng. Mỗi rule = 1 unit testable, kết quả: PASS / FAIL / INSUFFICIENT_DATA.
2–3 tuần
Soft Rule & Scoring Logic
Rule
Các rule tham khảo có trọng số: thâm niên công tác (càng cao càng tốt), mức lương so với ngành, pattern dòng tiền ổn định, số lần tra cứu CIC gần đây, sản phẩm vay hiện hữu. Kết quả: điểm số weighted.
2 tuần
Traffic Light Classification
Process
🟢 Xanh: Tất cả hard rule PASS + soft score ≥ threshold → Auto-approve eligible.
🟡 Vàng: Hard rule PASS nhưng soft score biên → Route cho Credit Officer review.
🔴 Đỏ: Bất kỳ hard rule FAIL hoặc fraud flag → Auto-reject hoặc escalate Specialist.
1–2 tuần
Rule Admin UI
Infra
Giao diện quản trị cho team Risk/Compliance: thêm/sửa/disable rule mà không cần deploy code. Version control cho rule changes, audit log ai sửa gì lúc nào. A/B testing rule mới trước khi apply toàn bộ.
2–3 tuần
📦 Deliverables Phase 2
Rule Engine Service (API) với 100% rule đã mã hóa
Traffic Light Classification Logic
Auto-routing workflow (Xanh → auto, Vàng → officer, Đỏ → specialist)
Rule Admin UI cho Risk/Compliance
Unit test & regression test suite cho mọi rule
⚠️ Rủi ro cần lưu ý
Rule thay đổi thường xuyên theo chính sách NHNN / nội bộ → Rule Engine phải thiết kế configurable, không hardcode. Ngưỡng traffic light (threshold) ban đầu cần calibrate cẩn thận với data lịch sử — nếu set quá chặt sẽ reject nhiều hồ sơ tốt, quá lỏng sẽ để lọt rủi ro.
🧠
Phase 3 — AI Risk Scoring & Fraud Detection
Train ML model dự đoán rủi ro vỡ nợ trên data lịch sử, xây module phát hiện giả mạo hồ sơ, anomaly detection cho các pattern bất thường.
8–10 tuần ML Engineer + Data Scientist + Risk Advanced
Credit Risk Scoring Model
AI
Train model (XGBoost / LightGBM) trên data lịch sử vay: features gồm thu nhập, DTI, thâm niên, CIC score, dòng tiền, ngành nghề, vùng miền. Target: dự đoán xác suất default trong 12 tháng. Model cần explainable (SHAP values) để compliance chấp nhận.
4–5 tuần
Document Fraud Detection
AI
Phát hiện giả mạo: ảnh CMND bị chỉnh sửa (Photoshop detection), bảng lương giả (font/layout bất thường), sao kê NH bị sửa số (pixel analysis). So sánh với known fraud patterns, flag confidence score cho mỗi document.
3–4 tuần
Anomaly Detection Pipeline
AI
Phát hiện pattern bất thường: thu nhập tăng đột biến trước khi vay, nhiều đơn vay cùng lúc ở nhiều tổ chức, dòng tiền "đẹp" bất thường (income smoothing giả). Unsupervised learning (Isolation Forest) + rule-based kết hợp.
2–3 tuần
Explainability & Compliance Dashboard
Infra
Dashboard cho Credit Officer / Auditor: hiển thị lý do cụ thể AI đánh giá Xanh/Vàng/Đỏ, top features ảnh hưởng quyết định (SHAP), so sánh với nhóm khách hàng tương đồng. Đảm bảo tuân thủ yêu cầu giải trình của NHNN.
2–3 tuần
📦 Deliverables Phase 3
Credit Risk Scoring Model (AUC-ROC ≥ 0.80)
Fraud Detection Module
Anomaly Detection Pipeline
Model Explainability Dashboard (SHAP-based)
Model validation report & backtest results
Model monitoring & retraining SOP
⚠️ Rủi ro cần lưu ý
Data lịch sử có thể biased (chỉ chứa hồ sơ đã được approve → survivorship bias). Cần kỹ thuật xử lý class imbalance (SMOTE, undersampling). Model AI phải explainable — black-box model sẽ không được NHNN / Compliance chấp nhận. Fraud detection cần cập nhật liên tục vì kỹ thuật giả mạo ngày càng tinh vi.
🚀
Phase 4 — UAT, Go-Live & Vận hành liên tục
Testing toàn diện, A/B test so sánh với quy trình cũ, training nhân viên, go-live từng bước, thiết lập monitoring & continuous improvement loop.
4–6 tuần Toàn bộ team + Credit Ops + Training Go-Live
UAT & Parallel Run
Process
Chạy song song hệ thống mới vs cũ trong 2-3 tuần: cùng 1 hồ sơ, so sánh kết quả. Tracking: tỷ lệ khớp quyết định, thời gian xử lý, số trường hợp cần human override. Đây là bằng chứng thuyết phục nhất cho management.
2–3 tuần
Training & Change Management
Process
Đào tạo theo role: Credit Officer (cách review hồ sơ Vàng), Specialist (fraud investigation workflow), Manager (dashboard & override), Risk/Compliance (rule admin). Kèm video hướng dẫn, FAQ, và sandbox environment để thực hành.
1–2 tuần
Phased Go-Live
Infra
Go-live theo từng bước: Tuần 1 — 1 chi nhánh pilot (10% volume). Tuần 2-3 — mở rộng 30% volume. Tuần 4+ — rollout toàn bộ. Mỗi bước đều có kill switch để rollback nếu phát hiện vấn đề nghiêm trọng.
2–4 tuần
Monitoring & Continuous Improvement
Infra
Dashboard realtime: OCR accuracy, rule pass/fail rates, model performance drift, processing time SLA. Alert tự động khi accuracy drop hoặc anomaly spike. Monthly review cycle: tune model, update rules, retrain nếu cần.

⚠️ Rủi ro cần lưu ý
Resistance to change từ nhân viên lâu năm — cần chứng minh hệ thống hỗ trợ họ (giảm việc nhàm chán) chứ không thay thế họ. Model drift theo thời gian (thị trường thay đổi, khách hàng thay đổi) → cần retraining schedule cố định. Quy định NHNN về AI trong tín dụng đang tiến triển — cần giám sát và comply kịp thời.
